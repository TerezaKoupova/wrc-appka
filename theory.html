Když chceš všechno najednou, nedostaneš nic pořádně.

PROMPT · 500+ slov „Potřebuju appku pro náš tým, je nás 8 lidí a furt si přehazujeme úkoly v Slacku a nikdo neví co je hotový a co ne. Tak jsem si říkal že by bylo dobrý mít nějakou appku kde by se to dalo trackovat. Chtěl bych tam auth přes Google a GitHub, ať se nemusí registrovat, a taky real-time sync, protože když kolega označí úkol jako hotový tak to chci vidět hned bez refreshe. A kanban board by byl super, něco jako Trello, s drag & drop. Vlastně by to chtělo i offline mode, protože někdy pracuju ve vlaku. A víš co, přidej tam AI — ať mi to automaticky sumarizuje co tým udělal za týden. Dark mode samozřejmě, tagy, prioritu 1–5, deadline notifikace na email, mobile push notifikace, a ideálně i desktop klient v Electronu. Export do PDF/CSV/iCal, Stripe subscription se třemi tiery (free, pro, enterprise), A/B testing, Mixpanel analytics, keyboard shortcuts, undo/redo, sharing přes veřejné linky, OAuth SSO, webhook integrace, CLI nástroj, browser extension, Slack & Teams notifikace…"

* PRD = Product Requirements Document — co se má postavit, pro koho a proč. Zadání pro AI místo pro tým.

PRD.md
# Problém
Tým 8 lidí si přehazuje úkoly v Slacku.
Nikdo neví, co je hotové.

# Uživatel
Projekt manažer v malém studiu.
Otevře appku 5× denně.
Kolega — vidí své úkoly, označí hotovo.

# Top 3 akce
1. Přidat úkol
2. Označit hotovo
3. Přiřadit kolegu

# Scope IN
Přidat úkol, označit hotovo, přiřadit
Filtrování podle stavu

# Scope OUT
Notifikace, real-time, offline mode

# Data model
tasks (id, title, status, owner_id)
users (id, name, avatar)

Co správný PRD obsahuje — a co pro workshop stačí.

PRD by-the-book

  • Problém a kontext
  • Cílový uživatel
  • Cíle a metriky úspěchu
  • User stories / use cases
  • Scope (in / out)
  • Funkční požadavky
  • Nefunkční požadavky
  • Datový model
  • Závislosti a rizika
  • Časový rámec

AI díky PRD trefil první řešení. Se SDLC (Software Delivery Life Cycle) to můžeš dál rozvíjet a nerozbít.

Naivní vibe coding

  • Náhodně střílím nápady
  • Push na main a modlím se
  • Klíče v kódu
  • SQL z hlavy
  • Funguje mi to lokálně
  • Náhodné díry v kódu
vs

SDLC workflow

  • Issue-driven development
  • Otestuju si před releasem
  • .env.local + .env.example
  • Migrace v gitu
  • Preview URL jako staging
  • Code review před mergem
